# VEDA Create Project Workflow

## Purpose

This document defines the structured workflow used to create and initialize a new project inside VEDA.

The workflow ensures that every project begins with a consistent lifecycle and architectural foundation.

Projects are treated as **brand ecosystem containers** that evolve through a defined lifecycle.

All mutation follows the rule:

Propose → Review → Apply

---

## Workflow Overview

Project creation follows a predictable sequence.

1. Create project container
2. Draft project blueprint
3. Review blueprint
4. Apply blueprint
5. Begin keyword research
6. Define keyword targets
7. Begin SERP observation

Each stage aligns with the project lifecycle states defined in the roadmap.

---

## Step 1 — Create Project Container

The operator creates the base project record.

Minimum input:

- project name

System result:

- project container created
- lifecycle state: `created`

No surfaces or keywords exist yet.

---

## Step 2 — Draft Project Blueprint

An initial blueprint is proposed for the project.

Blueprint proposal may be generated by an LLM assistant.

The blueprint describes:

- brand identity
- strategic niche
- surface registry
- website architecture model
- content archetypes
- entity clusters
- keyword territory
- authority model

Lifecycle state:

`draft`

---

## Step 3 — Blueprint Review

The operator reviews the proposed blueprint.

Possible actions:

- approve blueprint
- request revision
- modify blueprint

LLM systems may assist in editing proposals but cannot apply them.

---

## Step 4 — Apply Blueprint

Once approved, the blueprint becomes the structural contract for the project.

System results:

- blueprint stored
- declared surfaces activated

Lifecycle remains:

`draft`

until keyword research begins.

---

## Step 5 — Begin Keyword Research

Research identifies the territory the project will target.

Inputs:

- seed keywords
- competitor domains
- entity clusters

Outputs:

- keyword discovery pool
- topic clusters

Lifecycle transition:

`draft` → `researching`

---

## Step 6 — Define Keyword Targets

The operator selects a subset of discovered keywords to track.

Typical target set:

50–200 keywords.

System result:

- `KeywordTarget` records created

Lifecycle transition:

`researching` → `targeting`

---

## Step 7 — Begin SERP Observation

SERP snapshots begin collecting historical data.

System results:

- `SERPSnapshot` records
- SERP deltas
- volatility signals

Lifecycle transition:

`targeting` → `observing`

The project now participates fully in the VEDA observatory system.

## Key Principles

The workflow is designed to ensure:

- repeatable project creation
- controlled mutation
- deterministic system behavior
- clear operator oversight

Each project follows the same initialization path before entering long-term observation and development.
