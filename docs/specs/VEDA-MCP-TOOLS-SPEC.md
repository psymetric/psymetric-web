# VEDA MCP Tools Specification

## Purpose

This document defines the Model Context Protocol (MCP) tools required for LLM assistants to interact safely with VEDA.

MCP tools provide a controlled interface between LLM systems and the VEDA HTTP API.

LLMs may read system state and propose actions through these tools. They may not mutate system state without explicit operator confirmation.

All mutation must follow:

Propose → Review → Apply

---

## Design Principles

MCP tools must follow these rules:

- API-only interaction
- deterministic outputs
- project-scoped operations
- no silent mutation

All project-scoped tools must resolve project scope using:

resolveProjectId()

---

## Core MCP Tool Groups

### Project Management

Tools responsible for project creation and configuration.

Proposed tools:

create_project
get_project
list_projects
update_project_metadata

These tools manage project containers only. They do not create blueprint data automatically.

---

### Project Blueprint Tools

Tools used to propose and manage project blueprints.

Proposed tools:

propose_project_blueprint
get_project_blueprint
apply_project_blueprint
revise_project_blueprint

Blueprint proposals may be generated by LLM assistants but require operator approval before application.

---

### Keyword Research Tools

Tools supporting discovery and clustering of keyword opportunities.

Proposed tools:

run_keyword_discovery
cluster_keyword_candidates
get_keyword_candidates

These tools produce candidate keywords only.

They must not create `KeywordTarget` records automatically.

---

### Keyword Target Tools

Tools used to manage tracked keyword targets.

Proposed tools:

create_keyword_targets
list_keyword_targets
get_keyword_target
remove_keyword_target

Creation of keyword targets must always be an explicit operator decision.

---

### SERP Observation Tools

Tools that trigger or retrieve SERP observation data.

Proposed tools:

run_serp_snapshot_ingest
get_serp_snapshots
get_serp_deltas
get_keyword_volatility

These tools expose the SERP observatory surfaces.

---

### Observatory Intelligence Tools

Tools that retrieve computed intelligence surfaces.

Proposed tools:

get_project_volatility_summary
get_volatility_alerts
get_operator_briefing
get_keyword_impact_ranking
get_operator_action_hints

These tools are read-only.

---

## Relationship To System Layers

MCP tools act as a gateway to the VEDA architecture stack.

```
LLM Assistant
      ↓
MCP Tools
      ↓
HTTP API
      ↓
VEDA System Layers
```

This separation ensures assistants remain outside the core system.

---

## Future MCP Tool Groups

Future expansions may include tools for:

- social surface observatories
- content graph management
- execution planning
- SEO lab experimentation

These tools will follow the same deterministic and proposal-driven interaction model.

---

## Non Goals

MCP tools must not:

- access the database directly
- bypass API validation
- perform hidden mutations
- bypass project isolation

The MCP layer is an interface, not a control plane.
