# DQ-001 — Topic proposals deferred from Phase C1

## Status
Deferred

## Area
SERP-to-Content-Graph proposal helpers

## Decision
Topic proposals are not included in Phase C1.

## Reason
- Phase C1 should stay narrow and mapped-page oriented.
- Archetype and schema proposals are more concrete, lower-ambiguity first proposal surfaces.
- Topic proposals introduce more planning semantics and broader coverage interpretation than the first slice should carry.

## Next Eligible Phase
C1B or C2

## Blocking / Unlock Conditions
- The initial proposal endpoint contract must stabilize first.
- Archetype and schema proposal outputs must be implemented, hammered, and proven stable.
- Operator review surfaces should validate that the proposal model is useful before broader topic-level proposal semantics are added.
